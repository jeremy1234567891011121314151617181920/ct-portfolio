Reflection

The Process

When I first started creating this website, I learnt all the different components of HTML and CSS. I learnt that HTML was the structure and CSS was the styling. As I learnt more and more, I started to learn the different elements such as h1, p, div, and a. I also learnt how to link different pages together using the a element, allowing users to press on them to lead them to other links, making it possible to direct them to my GitHub profile and other projects such as this one right here. It has gotten to this point where my website has complex components such as a gradient background utilizing the colours blue and cyan, an image and several clickable links. Here are the key decisions, challenges and the skills/ technologies I used to create this website. Enjoy!

Key Decisions

- One of the key decisions I made were constructing a clear image of what my website would look like, and I am proud of what I have ended up with.

- I also decided to use a gradient background, which I think adds a nice touch to the website, making it look more aesthetically pleasing and more unique.

- Another key decision that I don't regret making was to include a profile image, showing off my personality and adding a direct link to my GitHub profile, which you guys can access from the top left corner of my website.

- Finally, I decided to make the website responsive, ensuring it looks good on all devices. This allows all devices to be able to look at it clearly, without making it too messy. Phones, computers, smart TVs, tablets, they ALL work!

Challenges

- To me personally, one of the main challenges I faced was correctly placing each and every text, image, box in its allocated location. It took me a while to find out and get used to making the image in the center as well as making the texts go to specific sides.

- Another challenge I faced was making the colour of my website gradient. I had to learn how to use the linear-gradient function, and it took me a while to understand how it worked and how to use it. I also had to experiment with different colours and angles to get the desired effect. Even though it took a very long time, I don't regret it a single bit as my website turned out great!

- Finally, my mind had a TON of different ideas for the structure of my website. It was so hard to choose!. I decided to stick with a simple, but detailed design after my teacher recommended doing so because it would be too complicated for an amateur coder like me to create such a complex structure.

Skills/ Technologies Used

I used a variety of different technologies and skills to create this website. These included, HTML, CSS, responsive design, Visual Studio Code, Git and GitHub.

- I used HTML to create the structure and content of the website. Using the different elements and tags allowed me to create an organised and interactive site.

- I used CSS to style the website and make it visually appealing, inclyding gradient backgrounds, box outlines, images, different fonts, text sizes, and colour.

- I used responsive design to ensure that all devices could access my website so it looks organised and easy to navigate.

- I used Visual Studio Code as the heart of my construction, where I typed up everything on HTML and CSS to create the website.

- Finally, I used Git and GitHub to keep track of all the changes I made to my website, committing each edit I made so anyone can check my profile and see the process of the website's creation.
